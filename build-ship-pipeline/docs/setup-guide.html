Quickstart (Docker Compose)

The fastest way to run the full stack including Postgres, OTel, Grafana, and the UI.

1

Clone and enter the project

git clone https://github.com/yaronyudov/CodeDelivery.git
cd CodeDelivery/build-ship-pipeline
2

Create your .env file

# .env — copy this block and fill in the values

# Required: JWT signing key (generate with: openssl rand -hex 32)
SECRET_KEY=your-secret-key-here

# Required: initial admin account (seeded to DB at first startup)
SEED_USERNAME=admin
SEED_PASSWORD=changeme-in-production

# LLM provider API keys (at least one required)
ANTHROPIC_API_KEY=sk-ant-...
# OPENAI_API_KEY=sk-...
# GROQ_API_KEY=gsk_...

# PostgreSQL (matches docker-compose defaults — change in production)
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DB=build_ship
POSTGRES_USER=pipeline
POSTGRES_PASSWORD=pipeline_secret

# CORS (leave empty for same-origin production; set for local dev)
# ALLOWED_ORIGINS=http://localhost:5173,http://localhost:8080
3

Start everything

docker compose up -d

This starts Postgres, the OTel collector, Prometheus, Loki, Tempo, Grafana, the UI backend, and nginx. The ui service automatically runs python -m ui.backend.seed on first start.

4

Open the UI

Navigate to https://localhost (self-signed cert) or http://localhost:8080 (direct). Log in with SEED_USERNAME / SEED_PASSWORD.

✅
You should see the login page, then after login: the run history sidebar + feature input. Type a feature request and click Execute.

Local Dev (without Docker)

1

Install Python dependencies

pip install -e ".[dev]"
2

Apply the DB schema

# Start Postgres (e.g. via Docker)
docker run -d --name pg -e POSTGRES_DB=build_ship \
  -e POSTGRES_USER=pipeline -e POSTGRES_PASSWORD=pipeline_secret \
  -p 5432:5432 postgres:16

psql postgresql://pipeline:pipeline_secret@localhost:5432/build_ship \
  -f src/db/schema.sql
3

Seed the database

SEED_USERNAME=admin SEED_PASSWORD=changeme python -m ui.backend.seed

This creates the admin user and inserts the 6 built-in system skills. Idempotent — safe to run multiple times.

4

Build the frontend

cd ui/frontend
npm install
npm run build
cd ../..
5

Start the backend

SECRET_KEY=$(openssl rand -hex 32) \
POSTGRES_HOST=localhost \
ANTHROPIC_API_KEY=sk-ant-... \
uvicorn ui.backend.app:app --port 8080 --reload

Configuration Files

config/budget.yaml — cost ceilings

Controls how much money/tokens a single pipeline run may consume. Applied globally to every run.

budget:
  # Global ceilings for the entire pipeline run
  tokens_limit: 2_000_000       # total tokens across all agents
  cost_limit_usd: 5.00          # USD hard ceiling per run
  steps_limit: 120              # max number of agent calls

  # Hard ceilings per single LLM call (never exceeded regardless of dynamic caps)
  hard_action_token_ceil: 60_000
  hard_action_cost_ceil_usd: 0.50

  # Debugger loops back to coder this many times before escalating to planner
  debug_max_attempts: 3
ℹ️
The governance layer enforces dynamic caps: as cost_used_usd approaches cost_limit_usd, per-action limits tighten proportionally. The pre-flight budget check happens before every LLM call, so the ceiling can never be exceeded.

config/prices.yaml — model pricing and assignments

Controls which model each agent uses by default and how much each model costs (used for pre-flight estimation and budget reconciliation).

# USD per 1K tokens — update when providers change pricing
prices:
  claude-sonnet-4-6:
    in: 0.003
    out: 0.015
  claude-haiku-4-5-20251001:
    in: 0.00025
    out: 0.00125

# Default model per agent role
agent_models:
  planner: claude-sonnet-4-6    # needs deep reasoning
  coder: claude-sonnet-4-6      # needs reliable code generation
  docker: claude-haiku-4-5-20251001  # simpler task, cheaper model
  security: claude-sonnet-4-6  # needs security expertise
  # ... (others default to haiku)

# Expected output tokens per agent (used in pre-flight cost estimation)
expected_output_tokens:
  planner: 2000
  coder: 4000
⚠️
The model a user selects in the UI overrides agent_models — that model is used for all agents. Per-agent model assignment only applies when no per-run override is set.

Defining Skills

Skills let you change what every pipeline run does without touching code. There are two kinds:

prompt_injection

Appends prompt_addon text to an agent's system prompt. Great for:

  • Technology stack requirements ("always TypeScript")
  • Security checklists ("check OWASP Top 10")
  • Code style rules ("use Google style guide")
  • Framework preferences ("use FastAPI, not Flask")

agent_toggle

Removes an agent from the run entirely (zero LLM cost). Great for:

  • Skipping Docker if you don't containerise
  • Skipping observability in quick prototypes
  • Skipping coverage review for spike/throwaway code

Note: planner and coder are protected — they cannot be disabled.

Creating a skill via the UI

  1. Log in to the pipeline UI
  2. Click the ⚙ Settings icon at the bottom of the sidebar
  3. Click New Skill in the SkillsManager panel
  4. Fill in the fields (see table below) and click Create
  5. Optionally toggle Default to apply it to every future run automatically

Creating a skill via the API

# First, get a session cookie by logging in
curl -c cookies.txt -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"changeme"}'

# Create a prompt_injection skill for the coder agent
curl -b cookies.txt -X POST http://localhost:8080/api/skills \
  -H "Content-Type: application/json" \
  -d '{
    "id": "use-typescript",
    "name": "Use TypeScript",
    "description": "Always write TypeScript with strict mode",
    "kind": "prompt_injection",
    "target_agents": ["coder"],
    "prompt_addon": "Always write TypeScript (never plain JavaScript). Enable strict mode. Avoid any type.",
    "is_default": true
  }'

# Create an agent_toggle skill to disable security review
curl -b cookies.txt -X POST http://localhost:8080/api/skills \
  -H "Content-Type: application/json" \
  -d '{
    "id": "skip-security-review",
    "name": "Skip Security Review",
    "kind": "agent_toggle",
    "target_agents": ["security"],
    "is_default": false
  }'

# Make a skill the default (or un-default it)
curl -b cookies.txt -X PATCH http://localhost:8080/api/skills/use-typescript/default

# List all skills
curl -b cookies.txt http://localhost:8080/api/skills

Per-session skill overrides

When starting a run, you can override the global defaults per-agent. This is exposed in the UI via the Session Skills panel (collapsed under the feature textarea).

Via API, send skill_overrides in the start run body:

# Start a run with skill overrides
curl -b cookies.txt -X POST http://localhost:8080/api/runs \
  -H "Content-Type: application/json" \
  -d '{
    "feature_request": "Build a REST API for a todo app with authentication",
    "model_config": {"provider": "anthropic", "model": "anthropic/claude-sonnet-4-6"},
    "require_approval": false,
    "skill_overrides": {
      "*": {
        "add": ["use-typescript"],    // add to ALL agents
        "remove": []
      },
      "coder": {
        "add": ["fastapi-style"],     // add only to coder
        "remove": []
      },
      "docker": {
        "add": [],
        "remove": []                  // empty = no change vs defaults
      }
    }
  }'
ℹ️
The "*" key applies to all agents. Per-agent keys are merged on top. remove can disable a default skill for this run only without permanently changing the global default.

Changing default models per agent

Edit config/prices.yaml → agent_models section. The server reads this at startup — restart after changes.

agent_models:
  planner: openai/gpt-4o           # use GPT-4o for planning
  coder: anthropic/claude-sonnet-4-6  # keep Sonnet for coding
  security: groq/llama-3.1-70b-versatile  # cheap model for review
⚠️
Add the model's prices to the prices section too, or budget estimation will fail. Keys must match exactly what LiteLLM uses.

Using Ollama (local models)

  1. Install Ollama: ollama.ai
  2. Pull a model: ollama pull llama3
  3. In the UI ModelPicker, select Ollama, set model to ollama/llama3, and set API base to http://localhost:11434
  4. No API key needed

Production Checklist

⚠️
Never use the Docker Compose defaults (pipeline_secret) in production. Change all passwords and secrets.

Security hardening

  • Generate a strong SECRET_KEY: openssl rand -hex 32
  • Use a strong SEED_PASSWORD (min 16 chars, mixed case + symbols)
  • Set POSTGRES_PASSWORD to something unique, not pipeline_secret
  • Leave ALLOWED_ORIGINS empty in production (nginx serves everything from the same origin)
  • Put the app behind nginx with TLS (already configured in ui/nginx.conf)
  • Restrict Postgres to localhost / internal network only

Updating model prices

When providers change pricing, update config/prices.yaml and restart. The budget ledger uses estimated costs at call time — old rows are not retroactively updated.

Grafana dashboards

Available at http://localhost:3000 (admin/admin by default). Three pre-built dashboards:

  • Pipeline Overview — run counts, success rate, total spend
  • Agent Performance — per-agent latency, token usage, cost
  • Budget Alerts — runs approaching or exceeding ceilings

Verifying everything works

# Run unit tests (no LLM calls needed)
pytest tests/unit/ -v

# Check the DB has the 6 system skills
psql $DATABASE_URL -c "SELECT id, kind, is_default FROM skills;"

# Verify the API is up
curl -s http://localhost:8080/api/auth/me | python -m json.tool
✅
If pytest shows 16/16 passed, SELECT returns 6 skills, and /api/auth/me returns a 401 (unauthenticated — expected), the system is healthy.
